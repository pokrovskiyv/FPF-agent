# enrich_metadata

> Источник: `scripts/enrich_metadata.py`

## Назначение

Добавляет в записи метаданных пользовательские ключевые слова и запросы на обычном языке (EN + RU) там, где автоматическое извлечение дало только жаргон. Оглавление часто содержит сугубо внутреннюю терминологию, а этот скрипт дополняет её формулировками реальных пользователей — вопросами вроде "How to choose between competing alternatives for a project?" или "Как сравнить два подхода?".

Запускается как постобработка `sections/metadata.json` и **идемпотентен** — повторный запуск не дублирует записи.

## Интерфейс

| Функция | Сигнатура | Что делает |
|---------|-----------|-----------|
| `main` | `() -> None` | Загружает `metadata.json`, проходит по списку `ENRICHMENTS` и мёржит новые ключевые слова и запросы в нужные записи |

На уровне модуля определён список `ENRICHMENTS` — ручной набор словарей с `pattern_id`, `add_keywords` и `add_queries` (двуязычные). Чтобы расширить список, редактируют этот объект напрямую.

## Алгоритм

1. Читаем `sections/metadata.json`.
2. Для каждой записи обогащения находим целевую запись по `pattern_id`; пропускаем, если не найдена.
3. Считаем разницу множеств между предлагаемыми и уже существующими ключевыми словами/запросами (защита от дублей).
4. Добавляем только новые элементы в поля `keywords` и `queries`.
5. Записываем результат обратно в `sections/metadata.json` с `ensure_ascii=False`.

Скрипт печатает итоговую строку по каждой обогащённой записи (`A.19.CPM: +5 keywords, +6 queries`), чтобы видеть, какие паттерны затронуты.

## Зависимости

**Импорты:** `json`, `pathlib.Path` — только стандартная библиотека.

**Где используется:** вызывается из `scripts/rebuild_all.sh` (шаг 3). Запускается после того, как [build_metadata](build_metadata.md) создал базовый файл.
