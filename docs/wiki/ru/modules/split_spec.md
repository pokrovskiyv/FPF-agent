# split_spec

> Источник: `scripts/split_spec.py`

## Назначение

Первый этап конвейера пересборки. Читает монолит `FPF-Spec.md` (5.5 МБ) и раскладывает его на ~240 файлов-секций, сгруппированных по Part и Cluster. Заголовки H1 становятся границами директорий, H2 — отдельными файлами. Для каждой директории генерируется `_index.md` со списком секций.

Модуль существует потому, что полный текст спецификации не помещается в контекстное окно целиком. Всё дальнейшее (метаданные, маршруты, эмбеддинги) работает уже с декомпозированными файлами.

## Интерфейс

| Функция | Сигнатура | Что делает |
|---------|-----------|-----------|
| `slugify` | `(text: str, max_len: int = 60) -> str` | Переводит текст заголовка в ASCII-слаг, безопасный для файловой системы |
| `extract_pattern_id` | `(heading: str) -> str` | Извлекает иерархический идентификатор паттерна (например, `A.6.P` или `B.3.3`) |
| `is_real_h1` | `(line: str) -> bool` | Отсекает ложные H1 — строки таблицы, начинающиеся с `#` |
| `is_h2` | `(line: str) -> bool` | Проверка на `## ` в начале строки |
| `parse_spec` | `(spec_path: Path) -> list[dict]` | Потоковый парсер: выдаёт вложенную структуру H1 → H2 с содержимым |
| `write_sections` | `(sections: list[dict], output_dir: Path) -> dict` | Создаёт директории, пишет секции и `_index.md` |
| `main` | `() -> None` | Точка входа с argparse (`--spec`, `--output`) |

## Алгоритм

1. Читаем `FPF-Spec.md` построчно.
2. На каждом H1 закрываем предыдущую директорию и открываем новую с числовым префиксом (`04-part-a-...`).
3. На каждом H2 закрываем предыдущий файл и открываем новый; если в заголовке есть пиковый ID, он добавляется в префикс имени.
4. Прочие строки накапливаются в буфер текущего H2 (или в преамбулу H1, если H2 ещё не начинался).
5. После последней строки дозаписываем последний H2 и сохраняем все файлы.
6. Для каждой директории создаём `_index.md` со списком ссылок на секции и их pattern ID в скобках.

Особые случаи: `is_real_h1` отсекает строки таблиц, а заголовки короче 3 символов игнорируются.

## Зависимости

**Импорты:** `argparse`, `re`, `unicodedata`, `pathlib.Path` — только стандартная библиотека.

**Где используется:** вызывается как подпроцесс из `scripts/rebuild_all.sh` (шаг 1 из 8). Другими модулями не импортируется.
