# test_smoke

> Источник: `scripts/test_smoke.py`

## Назначение

Дымовые тесты целостности конвейера, работающие без внешних зависимостей. Проверяют, что `metadata.json` парсится корректно, все разрешённые пути к файлам существуют, все 10 файлов маршрутов на месте с минимальной длиной цепочки, в таблице глоссария минимум 40 записей, а перекрёстные ссылки указывают на существующие pattern ID. Отдельный флаг `--all` включает более строгий набор, запускающий семантический поиск через `uv` как подпроцесс.

Запускаются после каждой пересборки для раннего обнаружения регрессий.

## Интерфейс

Модуль использует `unittest`. Каждый публичный класс автоматически обнаруживается `unittest.main`:

| Класс | Что проверяет |
|-------|--------------|
| `TestMetadataIntegrity` | metadata.json существует и парсится, ≥230 записей, нет жирных маркеров в ключах, у каждой записи есть title, все разрешённые пути существуют (≥200 разрешено) |
| `TestRouteFiles` | все 10 файлов маршрутов есть, упомянутые файлы секций существуют, у каждого маршрута ≥1 core секции и ≥3 элементов цепочки |
| `TestGlossary` | `glossary-quick.md` существует, имеет заголовок `| Term |`, ≥40 строк данных |
| `TestCrossReferences` | минимум 5 файлов `_xref.md`, неразрешённых pattern ID не более 20% от общего числа ссылок |
| `TestSemanticSearch` | только при `--all`: топ-скор ≥ 0.35, релевантные PID попадают в top-3 для калибровочных запросов |

Хелпер: `load_metadata() -> dict` — общее чтение JSON для всех сьютов.

## Алгоритм

`unittest.main` автоматически находит и запускает тестовые классы. Флаг `RUN_ALL = '--all' in sys.argv` включает семантический сьют через `@unittest.skipUnless(RUN_ALL, ...)`. Внизу модуль вычищает `--all` из `argv` перед передачей в unittest, иначе тот выдаст ошибку о неизвестном аргументе.

## Зависимости

**Импорты:** `json`, `os`, `re`, `subprocess`, `sys`, `unittest`, `pathlib.Path` — только стандартная библиотека для базового сьюта; семантический сьют вызывает `uv` как подпроцесс.

**Где используется:** запускается отдельно (`python3 scripts/test_smoke.py`); документирован как шаг проверки в CLAUDE.md и в `rebuild_all.sh` (хотя shell-скрипт сам не вызывает — тесты запускаются вручную).
