# Трёхъярусная загрузка

## Компоненты

Загрузка секций разделена на три яруса, балансирующих точность и вычислительные затраты. Классификатор выбирает ярус, ретривер его исполняет.

| Ярус | Когда срабатывает | Что грузит | Типичный бюджет |
|------|-------------------|------------|----------------|
| **Tier 1 — Маршрут / поиск термина** | Burden совпал с одним из 10 маршрутов, или пользователь назвал pattern ID | Подобранная цепочка секций (маршрут) или одна секция (term lookup) | 800–1500 токенов |
| **Tier 2 — Семантический откат** | Сигнал есть, маршрут не совпал | Динамическая цепочка: keyword search + FAISS-семантика | 2000 токенов (включая ревьюера) |
| **Tier 3 — Комбинированный** | Совпадают несколько маршрутов (пересекающая задача) | Core первичного маршрута + семантическое дополнение | 2500 токенов (включая ревьюера) |

## Поток данных

```
Решение классификатора
     │
     ├──► Tier 1 ─► Ретривер, Mode A
     │              (читаем routes/route-N.md → сначала core YES секции,
     │               потом остальные при необходимости; откат к _xref.md
     │               для расширения в другие Part при застревании)
     │
     ├──► Tier 2 ─► Ретривер, Mode B
     │              (keyword search по metadata.json  +
     │               uv run scripts/semantic_search.py "<SEARCH_QUERY>" --top-k 5 --json
     │               объединяем, дедуплицируем по pattern ID, оставляем score ≥ 0.45,
     │               сортируем по иерархии pattern ID)
     │
     └──► Tier 3 ─► Mode A core + Mode B дополнение
                    (грузим core маршрута, потом шаги Mode B 1–5
                     по SEARCH_QUERY, дедуплицируем с уже загруженным)
```

## Решения

- **Маршруты как кэш, семантика как фундамент.** Маршруты дают детерминированную высококачественную загрузку для известных задач; семантика покрывает всё остальное. 10 маршрутов подобраны вручную из наблюдаемого использования и представляют эвристический кэш поверх 242 паттернов спеки.
- **Косинусная близость через нормализованный inner product.** `build_embeddings` нормализует вектора и строит `IndexFlatIP`, поэтому `index.search` возвращает эквивалент косинусного сходства. Порог 0.45 откалиброван по запросам в [test_smoke](../modules/test_smoke.md).
- **Иерархические pattern ID дают естественный порядок.** После объединения результатов ретривер сортирует по pattern ID (`A.6` перед `A.6.B`, `B.1` перед `B.1.3`), давая резонеру порядок от общего к частному.
- **Детекция застревания.** Если ретривер замечает циклы (одни и те же секции повторно), эскалирует до Tier 3, добавляет кросс-ссылки из `_xref.md` или сообщает пользователю.
- **Ревьюер только для Tier 2/3.** Маршруты Tier 1 предварительно провалидированы; шаблонной дисциплины резонера достаточно. Динамической загрузке нужна дополнительная проверка обоснованности.
