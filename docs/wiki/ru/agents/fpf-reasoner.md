# fpf-reasoner

> Источник: `agents/fpf-reasoner.md`

## Назначение

Третий агент и единственный, который разговаривает с пользователем. Читает FPF-паттерны, загруженные ретривером, применяет их внутреннюю структуру к конкретной ситуации пользователя и возвращает практический, применимый результат на языке пользователя. **Резонер никогда не использует терминологию FPF** — никаких pattern ID, "холонов", "bounded context", "U.whatever". В исходнике это называется "Принцип #0: контракт обычного языка".

Аналогия из исходника: резонер — это GPS. Внутри он использует алгоритм Дейкстры. Пользователю он говорит "поверните направо через 200 метров." Он не говорит "применяется алгоритм кратчайшего пути на взвешенном графе."

## Интерфейс

**Вход:** загруженные ретривером секции + исходный вопрос пользователя + метка задачи от классификатора.

**Выход:** структурированный ответ по одному из десяти шаблонов (по одному на маршрут) или универсальный шаблон `semantic`, когда маршрут не подходит.

## Шаблоны вывода

Всего 11 шаблонов: по одному на каждую задачу плюс универсальный семантический. У каждого — фиксированный скелет, который резонер наполняет содержимым, специфичным для ситуации пользователя:

| Задача | Скелет (уровень заголовков) |
|--------|----------------------------|
| `project_alignment` | Зоны ответственности → Потоки работы → Пробелы |
| `language_discovery` | Таблица терминов → Рекомендуемое действие |
| `boundary_unpacking` | Правила → Условия доступа → Обязанности → Необходимые доказательства |
| `comparison_selection` | Таблица критериев → Пробелы в данных → Рекомендация |
| `generator_portfolio` | Список подходов → Сравнительная таблица → Переиспользуемый шаблон |
| `rewrite_explanation` | Переписанный текст → Что сохранено / изменено |
| `ethical_assurance` | Карта конфликтов → Реестр предубеждений → Чек-лист аудита |
| `trust_assurance` | Уверенность по компонентам → Пробелы в доказательствах → Рекомендации |
| `composition_aggregation` | Диагностика → Карта зависимостей → План исправлений |
| `evolution_learning` | Карта цикла → Точка разрыва → План замыкания → Индикаторы здоровья |
| `semantic` (универсальный) | Ситуация → Ключевые инсайты → Рекомендации → Точки внимания |

## Всегда подгружает

Перед генерацией резонер всегда читает два внутренних помощника:

- `sections/glossary-quick.md` — для внутренней ориентации
- `sections/lexical-rules.md` — для молчаливого применения замен терминов (пользователю не показывается)

## Режим ведения по маршруту

Для запросов по маршруту резонер инструктируется не вываливать всё сразу, а вести анализ шаг за шагом. Начать с самого релевантного инсайта из первой core-секции, дополнить структурой следующей, закончить конкретными действиями.

## Место в конвейере

```
ретривер → [fpf-reasoner] → (опционально: ревьюер) → вывод
```

На Tier 1 вывод резонера идёт пользователю напрямую. На Tier 2/3 после него запускается ревьюер как страж жаргона и обоснованности.
