# fpf-retriever

> Источник: `agents/fpf-retriever.md`

## Назначение

Второй агент. По решению классификатора загружает минимальный набор секций для ответа пользователю. Работает в двух режимах:

- **Mode A (Tier 1, маршрут):** загружает подобранную цепочку из `sections/routes/route-*.md`. Самый дешёвый и качественный путь.
- **Mode B (Tier 2, семантика / Tier 3, комбинация):** ищет по ключевым словам в `metadata.json` плюс семантически через [semantic_search](../modules/semantic_search.md), объединяет результаты, дедуплицирует по pattern ID.

## Интерфейс

**Вход:** вывод классификатора (TIER, BURDEN, ROUTE, SEARCH_QUERY, BUDGET, SECTIONS).

**Выход:** конкатенированное содержимое секций с источниками:

```
SECTIONS_LOADED: [число]
TOTAL_LINES: [приблизительно]

--- Section: [pattern_id] ([путь_к_файлу]) ---
[содержимое секции]

--- Section: [pattern_id] ([путь_к_файлу]) ---
[содержимое секции]
```

## Стратегии загрузки

Mode A (Tier 1) проходит три подуровня по порядку, останавливаясь при достаточном покрытии:

1. **Прямой поиск по pattern ID** — если пользователь назвал конкретный ID (A.6, E.17), читаем `metadata.json` и загружаем один файл.
2. **Загрузка цепочки маршрута** — читаем файл маршрута, сначала core-секции (отмеченные `YES`), потом остальные по порядку, пока вопрос не покрыт.
3. **Расширение через перекрёстные ссылки** — если покрытия не хватает, идём по `builds_on`/`prerequisite_for`/`coordinates_with` через файлы `_xref.md` в другие Part.

Mode B (Tier 2/3) — динамический конвейер из 5 шагов:

1. Поиск по ключевым словам в полях `keywords` и `queries` `metadata.json`, собираем top-10.
2. Запуск `uv run scripts/semantic_search.py "<SEARCH_QUERY>" --top-k 5 --json`, оставляем score ≥ 0.45.
3. Объединяем результаты поиска по ключам и семантики, дедуплицируем по pattern ID, максимум 5 секций.
4. Для top-3 смотрим `_xref.md` их директории и добавляем до 2 дополнительных секций по кросс-ссылкам.
5. Сортируем итог по pattern ID (иерархические ID дают естественный порядок от общего к частному).

Tier 3 совмещает оба режима: сначала core маршрута, потом дополнение через шаги Mode B по `SEARCH_QUERY`.

## Детекция застревания

Если ретривер замечает, что грузит одни и те же секции повторно или ходит по кругу, эскалирует до Tier 3 (кросс-ссылки), а если всё ещё тупик — сообщает пользователю: "Этот вопрос затрагивает несколько областей. Давайте расширим поиск." Может также загрузить `glossary-quick.md` как ориентир.

## Место в конвейере

```
классификатор → [fpf-retriever] → резонер
```

Ретривер передаёт резонеру полное содержимое секций — никогда не суммирует. Уважает бюджет из таблицы стратегий классификатора (800–2500 токенов в зависимости от тира).
