# fpf-reviewer

> Источник: `agents/fpf-reviewer.md`

## Назначение

Опциональный четвёртый агент, запускающийся на выводах Tier 2 (семантика) и Tier 3 (комбинированный), а также на пересекающих запросах Tier 1. Выполняет три проверки вывода резонера: страж жаргона (наивысший приоритет), валидация обоснованности и применимость. Если хоть одна проверка падает — ревьюер правит вывод на месте; если всё проходит — передаёт вывод резонера как есть.

Пропускается для простых поисков термина Tier 1 и чистых запросов по маршруту — они дёшевы, и шаблонной дисциплины резонера достаточно.

## Интерфейс

**Вход:** вывод резонера + исходные секции, которые загрузил ретривер.

**Выход:**

```
STATUS: [PASS | CORRECTED]
FIXES: [список исправлений, если были]

[финальный вывод для пользователя]
```

## Три проверки

### Check 1 — Страж жаргона (высший приоритет)

Сканирует вывод на любую FPF-специфичную терминологию, проскочившую наружу. В исходнике явно перечислены запрещённые термины: `holon`, `episteme`, `bounded context`, `transformer quartet`, `CharacteristicSpace`, `SenseCells`, `MVPK`, `Claim Register`, любые `U.*`, любые pattern ID (A.6, E.17, ...), аббревиатуры фреймворка (F-G-R, NQD, E/E-LOG, DRR, UTS, CSLC, USM, USCM), а также мета-ссылки вроде "according to FPF", "the framework", "the specification" и термины лексического долга `axis`/`dimension`/`metric` (как существительное). Любое совпадение запускает переписывание проблемного пассажа на обычный язык.

Примеры переписываний (из исходника):
- "Using U.Commitment deontic objects..." → "Here are the obligations this creates..."
- "The Boundary Norm Square suggests L/A/D/E routing..." → "This text mixes four different things: rules, conditions, obligations, and evidence requirements..."
- "Applying CharacteristicSpace A.19..." → "Here are the criteria to evaluate each option..."

### Check 2 — Валидация обоснованности

Каждое значимое утверждение должно прослеживаться до содержимого загруженных секций. Утверждения, вводящие концепции, отсутствующие в источниках, помечаются и либо удаляются, либо смягчаются.

- **Tier 2** — более строгая проверка, потому что цепочка секций собиралась динамически.
- **Tier 3** — смесь маршрутных и семантических секций: маршрутные утверждения — обычная проверка, семантические — по правилам Tier 2.
- Мосты между двумя секциями должны быть явно обоснованы.

### Check 3 — Применимость

Вывод должен быть конкретным (не общие советы), структурированным (таблицы/списки, не простыня текста), применимым (ясные следующие шаги) и лаконичным.

## Место в конвейере

```
резонер → [fpf-reviewer] → вывод    (только Tier 2, Tier 3, Tier 1 cross-cutting)
```
