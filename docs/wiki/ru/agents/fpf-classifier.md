# fpf-classifier

> Источник: `agents/fpf-classifier.md`

## Назначение

Первый агент конвейера. Берёт исходное сообщение пользователя и решает пять вопросов: есть ли вообще сигнал, который стоит структурировать; какой уровень использовать (маршрут / семантика / комбинация); какой конкретной задаче он соответствует; сколько токенов потратить; и какой поисковый запрос передать дальше. На выходе — структурированное решение маршрутизации. Ретривер и резонер не видят исходное сообщение, только вердикт классификатора.

Классификатор намеренно осторожен: при слабом сигнале возвращает `SIGNAL: no` и конвейер останавливается. Это защита от ложных срабатываний на обычных кодинг-вопросах.

## Интерфейс

**Вход:** сообщение пользователя на естественном языке.

**Выход:**

```
SIGNAL: [yes/no]
TIER: [1/2/3]
BURDEN: [тип_задачи]
CONFIDENCE: [HIGH/LOW/BYPASS]
ROUTE: [путь к файлу маршрута | "metadata.json" | null]
PIPELINE: [retriever→reasoner | retriever→reasoner→reviewer]
BUDGET: [бюджет токенов]
SECTIONS: [список файлов секций для предзагрузки]
SEARCH_QUERY: [запрос для семантического поиска при Tier 2/3]
```

## Логика классификации

Три этапа:

1. **Детекция сигнала** — ищем любой намёк на координацию, системную инженерию, междисциплинарность. Шире прямого совпадения: ловит "помоги мне структурировать", "всё в каше", "что я упускаю?". Отсекает: чистый код, вопросы о библиотеке/синтаксисе, файловые операции.
2. **Сопоставление маршрутов** — сверяем слова пользователя с таблицей триггеров для 10 известных задач (`project_alignment`, `language_discovery`, `boundary_unpacking`, `comparison_selection`, `generator_portfolio`, `rewrite_explanation`, `ethical_assurance`, `trust_assurance`, `composition_aggregation`, `evolution_learning`) плюс спецслучай `term_lookup`.
3. **Назначение тира** — одно сильное совпадение → Tier 1 (авто-диспатч). Несколько совпадений → Tier 3 (комбинированный). Сигнал есть, но совпадения нет → Tier 2 (семантический откат). Явно упомянут pattern ID (A.6, UTS, DRR) → обход уверенности, авто-диспатч Tier 1 `term_lookup`.

Таблица стратегий в исходнике сопоставляет каждую задачу с конкретным файлом маршрута, глубиной конвейера и бюджетом токенов.

## Место в конвейере

```
USER → [fpf-classifier] → ретривер → резонер → (ревьюер) → вывод
```

Все последующие агенты зависят от решения классификатора. Если `SIGNAL: no` — конвейер немедленно останавливается.
